# Maintainers

Nightgauge is stewarded by **Edibu, LLC** through the `nightgauge` GitHub
organization. Maintenance responsibilities are assigned by GitHub **team**, so
the authoritative, always-current roster is the team membership itself — this
file describes the roles those teams play.

| Team                                                                              | Responsibility                                                                                                                                 |
| --------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| [`@nightgauge/maintainers`](https://github.com/orgs/nightgauge/teams/maintainers) | Review, approve, and merge pull requests; triage issues; cut releases. Listed as the code owner in [`.github/CODEOWNERS`](.github/CODEOWNERS). |
| [`@nightgauge/security`](https://github.com/orgs/nightgauge/teams/security)       | Triage and coordinate vulnerability reports received via [SECURITY.md](SECURITY.md).                                                           |
| [`@nightgauge/bots`](https://github.com/orgs/nightgauge/teams/bots)               | Automation accounts (dependency updates, CI). Not human maintainers.                                                                           |

The project currently operates under a single-maintainer model; see
[GOVERNANCE.md](GOVERNANCE.md) for the decision model, the path to becoming a
maintainer, and continuity/succession.

## Contact

- **Security vulnerabilities:** do not open a public issue — follow
  [SECURITY.md](SECURITY.md).
- **Questions and support:** see [SUPPORT.md](SUPPORT.md) (GitHub Discussions).
- **Everything else:** open an issue or pull request.

## Contribution boundary

Before proposing a large feature, check
[GOVERNANCE.md § Project boundary](GOVERNANCE.md#project-boundary) and
[VISION.md](VISION.md) so your work lands within the public project.
