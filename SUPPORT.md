# Support

Nightgauge is an open-source project maintained by a single developer.
Support is real but best-effort — here is what to expect and where to go.

## Where to Ask

| Need                       | Channel                                                                     |
| -------------------------- | --------------------------------------------------------------------------- |
| Question / getting started | [GitHub Discussions](https://github.com/nightgauge/nightgauge/discussions)  |
| Bug report                 | [GitHub Issues](https://github.com/nightgauge/nightgauge/issues/new/choose) |
| Feature request            | [GitHub Issues](https://github.com/nightgauge/nightgauge/issues/new/choose) |
| Security vulnerability     | **Do not open a public issue.** See [SECURITY.md](SECURITY.md)              |

## What to Expect

- Issues and discussions are triaged **weekly**.
- There is no guaranteed response time — this is a solo-maintained project.
  Clear, reproducible reports get fixed fastest.
- Pull requests are welcome; see [CONTRIBUTING.md](CONTRIBUTING.md).

## Before Filing an Issue

1. Check [docs/TROUBLESHOOTING.md](docs/TROUBLESHOOTING.md) for known issues.
2. Search existing issues — duplicates are closed with a pointer to the
   original.
3. For pipeline problems, run `nightgauge doctor` and include the output,
   along with your OS, the component (VSCode extension / Go binary / SDK /
   skills), and the version (`nightgauge --version`).
