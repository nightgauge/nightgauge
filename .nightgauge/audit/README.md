# .nightgauge/audit/

This directory contains output from the `/nightgauge:product-audit` skill.

## Directory Structure

```
.nightgauge/audit/
├── README.md                        # This file
├── PRODUCT_AUDIT_REPORT.md          # Human-readable markdown report (latest run)
├── product-audit-latest.json        # JSON report (latest run, for CI/tooling)
├── product-audit-{timestamp}.json   # JSON report for the specific run
├── parity-config.json               # Feature parity matrix (managed by team)
├── REPORT_TEMPLATE.md               # Optional custom report template
└── history/
    ├── product-audit-2026-03-23.json
    ├── product-audit-2026-03-16.json
    └── ...
```

## File Descriptions

### Generated by skill (do not edit manually)

| File | Description |
| ---- | ----------- |
| `PRODUCT_AUDIT_REPORT.md` | Human-readable report for the most recent audit run. Updated on every run. |
| `product-audit-latest.json` | Machine-readable JSON for the most recent run. Used by CI and trend analysis. |
| `product-audit-{timestamp}.json` | JSON report for a specific run. Timestamp format: `YYYYMMDDTHHMMSSZ`. |
| `history/product-audit-{date}.json` | Historical archive. One file per calendar day. Trend analysis reads these. |

### Managed by team (commit to git)

| File | Description |
| ---- | ----------- |
| `parity-config.json` | Feature parity matrix definitions. Defines which features each client must implement. Edit this to add/remove features or clients. See [parity-config.json](parity-config.json) for schema. |
| `REPORT_TEMPLATE.md` | Optional custom Markdown template for report generation. If absent, the skill uses its built-in template. |

### Transient (cleaned up by skill after synthesis)

| File | Description |
| ---- | ----------- |
| `dimension-{1-8}.json` | Per-dimension raw findings written by parallel subagents. Deleted after Phase 3 (Synthesis) completes. Do not rely on these persisting. |
| `coverage-cache/` | Cached test coverage reports for `--quick` mode. Not committed to git. |

## Retention Policy

- **Latest reports** (`PRODUCT_AUDIT_REPORT.md`, `product-audit-latest.json`): Always present; overwritten on each run.
- **Timestamped reports** (root level): Kept only for the most recent run. Archived to `history/` automatically.
- **History**: Last 90 days by default. Configurable via `product_audit.reporting.report_retention_days` in `.nightgauge/config.yaml`.

## .gitignore Entries

```gitignore
# Product audit — generated reports and transient data

```

**Commit to git** (track in version control):

```gitignore
# These files define audit configuration — commit them
!.nightgauge/audit/parity-config.json
!.nightgauge/audit/REPORT_TEMPLATE.md
!.nightgauge/audit/README.md
```

## Running the Audit

```bash
# Full audit (report only)
/nightgauge:product-audit

# With issue creation for high+ severity findings
/nightgauge:product-audit --create-issues

# CI mode (exit 1 if score < 75)
/nightgauge:product-audit --ci --threshold 75

# Quick mode (use cached coverage, skip API probing)
/nightgauge:product-audit --quick
```

See `skills/nightgauge-product-audit/SKILL.md` for full CLI reference.

## Report Interpretation

### Score Ranges

| Score | Meaning |
| ----- | ------- |
| 90-100 | Excellent — all dimensions healthy |
| 75-89 | Good — minor issues only |
| 60-74 | Fair — some dimensions need attention |
| 45-59 | Poor — significant issues detected |
| 0-44 | Critical — immediate action required |

### Dimension Weights

Default weights (configurable in `.nightgauge/config.yaml`):

| Dimension | Weight | Rationale |
| --------- | ------ | --------- |
| API Alignment | 20% | Client-server mismatches break production features |
| Test Coverage | 20% | Coverage directly linked to production reliability |
| Security | 15% | Security issues have highest risk |
| Feature Parity | 15% | Feature gaps affect users across all clients |
| Lifecycle | 10% | Epic hygiene affects team velocity |
| Documentation | 10% | Doc drift slows developer onboarding |
| Dependencies | 5% | Background risk, self-correcting |
| CI/CD | 5% | Infrastructure health, self-correcting |
