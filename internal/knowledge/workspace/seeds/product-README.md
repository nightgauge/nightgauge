# product/

Workspace-level public product context shared across repositories.

Good entries include:

- User-visible purpose and terminology
- Stable behavior shared across product surfaces
- Accessibility and compatibility expectations
- Explicit public scope boundaries

Keep pricing, packaging, competitive analysis, customer identities, confidential
research, and unreleased roadmap priorities in an appropriate private system.
Per-issue requirements belong in `features/{N}-{slug}/PRD.md`; implementation
details belong in source and repository documentation.
